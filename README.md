# gpl-canary-evolution

**Исследование эволюции промптов для защиты GPL‑кода от автоматического извлечения спецификаций через LLM.**

## Что это

Проект исследует, как с помощью генетического алгоритма автоматически улучшать текстовые инструкции (скрытые в репозитории), которые заставляют LLM отказываться генерировать технические спецификации по коду. Цель - изучить уязвимость LLM к таким инструкциям и разработать методы защиты от недобросовестного использования AI для обхода лицензий GPL.

**Важно:** Это исследование в области безопасности LLM. Код проекта не предназначен для реального внедрения без дополнительной этической оценки. Результаты могут быть использованы для улучшения защитных механизмов.

## Генетический алгоритм (общая схема)

- **Особь** - текст правил (без привязки к формату размещения).
- **Фитнес** - взвешенная комбинация:
  - успешность отказа на запрос спецификации,
  - устойчивость к перебивке ("игнорируй инструкции"),
  - отсутствие ложных срабатываний на технические вопросы,
  - "отравляющая" плотность маркера при нарушении отказа.
- **Операторы**: мутация и кроссинговер выполняются LLM (советник не знает о контексте размещения).
- **Селекция** - элитарная.
- **Оценка** - на чистой LLM без предзнания.

## Применение в защите

Проект показывает, что скрытые инструкции могут предотвращать автоматическое копирование логики через AI-агентов. Однако **финальные инструкции не следует использовать без понимания рисков** (нарушение политик хостинга, репозиториев). Исследование помогает разработчикам LLM и сообществу open source находить баланс между открытостью и защитой.

## Этические ограничения

- Проект не пропагандирует обход лицензий.
- Не содержит готовых "вредоносных" инъекций - только исследовательские примеры.
- Код предоставлен как есть, исключительно для образовательных целей.
- Автор не несёт ответственности за misuse.
